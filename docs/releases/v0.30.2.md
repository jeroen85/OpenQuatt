# OpenQuatt v0.30.2

Release date: 2026-05-05

OpenQuatt `v0.30.2` is a stability and usability release. It improves the built-in web interface, adds trend and log diagnostics, reduces ESP32-S3 runtime pressure, improves Home Assistant cooling inputs, and adds initial Quatt Hybrid V2 support.

## Important Upgrade Note

This release introduces a new custom flash partition layout for flash-backed trend storage.

Devices currently running `v0.29.x` or older cannot install `v0.30.2` as their first upgrade via OTA. The first upgrade to `v0.30.2` must be done via USB using the factory image or the OpenQuatt webinstaller. After this one-time USB flash, future OpenQuatt updates can use OTA normally again.

## Highlights

- Added initial Quatt Hybrid V2 support, including model selection, V2 performance mapping, and updated power-limit handling.
- Added a Trends tab with local history for temperatures, power, efficiency, comfort, and flow.
- Added optional flash-backed trend history for longer history windows.
- Added a diagnostics log viewer with recent RAM log history and copy-to-clipboard support.
- Added settings backup and restore, including a comparison before applying restored settings.
- Improved dynamic Home Assistant inputs so ESPHome receives safe numeric values while separate validity flags protect control logic.
- Moved cooling dew point aggregation to Home Assistant; ESPHome now receives one selected dew point plus one validity signal.
- Enabled PSRAM on Waveshare ESP32-S3 builds and moved larger history buffers there when available.
- Reduced avoidable web, Modbus, OpenTherm, and diagnostic load.
- Updated ESPHome to `2026.4.3`.

## Breaking Or Behavioral Changes

- The firmware now uses a custom partition table with an `openquatt_data` partition for flash-backed trend storage. First upgrade from older releases requires USB/factory flashing.
- Cooling no longer depends on ESP-side room count or ESP-side per-room dew point calculation. The Home Assistant package now computes the highest valid cooling dew point.
- Obsolete cooling fallback diagnostics and per-room ESP cooling inputs were removed from the Home Assistant dashboard flow.
- Several diagnostic and status sensors now refresh less often. Control-critical signals remain fast; user-facing diagnostics are intentionally calmer.
- The project reverted to the core ESPHome web server after removing the experimental `web_server_idf` component.

## What Changed

### Web App And Diagnostics

- Improved first-load behavior so the loading modal stays visible until the overview has usable data.
- Split web polling into faster live data, slower bulk refreshes, and much slower static firmware refreshes.
- Limited live log streaming to the log modal only.
- Added recent RAM log history and longer log lines.
- Added settings backup export and restore.
- Added runtime web login support.

### Trends, Memory, And Runtime Load

- Added RAM-backed trend storage for short history windows.
- Added optional flash-backed trend storage using the new `openquatt_data` partition.
- Moved large trend/log buffers to PSRAM on Waveshare builds.
- Added internal-RAM fallback for heatpump-listener builds.
- Reduced stack and heap pressure in history endpoints.
- Reduced Modbus polling pressure and staged HP1/HP2 polling to avoid bursts.
- Bounded OpenTherm processing work per loop call.

### Home Assistant And Cooling

- Simplified dynamic Home Assistant value templates while keeping `entity_id|attribute_name` selector support.
- Added validity-aware dynamic HA inputs.
- Moved cooling dew point selection to Home Assistant.
- Updated HA dashboards for the new cooling and Quatt Hybrid V2 flows.
- Fixed HA template metadata compatibility and cooling safety icons.

### Quatt V2 And Control Behavior

- Added Quatt Hybrid V2 model selection and UI cards.
- Added V2 performance mapping and high-temperature continuation behavior.
- Derived duo power limits from mains current and scaled them for Quatt V2.
- Improved logging for ControlMode transitions, request decisions, compressor blocks, boiler/cooling guardrails, and runtime holds.
- Fixed scheduled OpenQuatt resume behavior and paused OpenQuatt bypass in Power House.
